# 基础设施部署方法论

## 渐进式搭建，不要一步到位

先跑通最简单的方案，遇到具体问题再加复杂度。

**反模式：** 一上来就设计 CF 代理 + HTTPS + IP 白名单 + 域名的完美方案
**正确做法：** HTTP 直连能跑吗？→ 需要 HTTPS？加证书 → 需要隐藏 IP？加 CF → 公司网络不通？换方案

每加一层都是因为解决了一个**具体问题**，而不是"可能需要"。

## 问题归因：先排除自身

网络不通时的排查顺序：
1. 自己的环境（VPN、代理、DNS 缓存）
2. 中间链路（CDN、NSG、防火墙）
3. 目标服务（进程、端口、配置）

**不要跳过第一步。** 今天误判"被墙"，实际是公司 VPN 的锅。

## 配置单一来源原则

同一个配置项只在一个地方定义。

- ✅ 环境变量统一放 `.bashrc`
- ❌ `.bashrc` 和 `settings.json` 都写 `ANTHROPIC_BASE_URL`

重复配置 = 迟早有一个过期，然后花 30 分钟排查"为什么不生效"。

## 记忆要精确，不要模糊

"旺财在某台 VM 上" → 错
"旺财在 VM1（4.186.30.196，印度浦那）" → 对

模糊的记录等于没记。关键信息写到能**无歧义还原**的程度。
