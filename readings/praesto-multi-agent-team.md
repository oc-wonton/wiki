# Praesto 多 Agent 团队实践笔记

> 来源: https://syndim-bot.github.io/Praesto/
> 团队: 猫窝 — 12 个 agent 跑在 OpenClaw 上，18 天从 0 到成型
> 阅读日期: 2026-04-15

## 核心发现

### 1. 每条规则来自一次失败
不是提前设计系统，而是翻车 → 复盘 → 变成规则文件。
"改一个文件 = 改行为" 是 agent 组织最大的优势。

### 2. 任务粒度是命门
编码时间从 44h → 13min，核心变化：每个任务只改 1 个文件。
- 粗粒度（改 4 个 service）→ 超时频发
- 细粒度（改 1 个文件）→ 超时消失

### 3. 角色定义三层演进
- V1: "你是架构师" → 输出泛泛
- V2: ANTI-PATTERNS（你不做什么）→ 不越界了但还是泛
- V3: 领域专家 Skill（78 文件 924KB 权威知识源）→ 有依据、有判断
**核心洞察：不是抽自己经验，是把顶级专家的思维框架给 agent 用**

### 4. Agent 审查不可信
"22/22 全绿" 可以是假阳性。应对：
- 三方交叉审查（产品 / 视觉 / 测试 并行）
- 协调者必须抽查
- 审查必须全量，禁止抽查

### 5. 三层审查（代码 / 渲染 / 操作）
Agent 不会在脑子里"渲染"。纯代码审查发现不了视觉问题。
需要截图 + 点按钮验证。

### 6. 工作流声明化
10 条 YAML 工作流用 / 命令触发。支持状态持久化 + 循环上限 + 自动升级。

### 7. workflow-retro 自动复盘
PR 提出时自动触发复盘，输出结构化报告，检查是否需要：
- 更新 WORKFLOWS.md
- 新增/改进 skill
- 同步更新规则文件
**把复盘变成代码级保证，不靠纪律性要求。**

### 8. 借鉴→落地流程
持续扫描 → 提取可借鉴点 → 建 Issue 跟踪 → 分派执行 → 验收闭环 → 后续迭代

## 我可以借鉴什么

### 立即可用
- **任务粒度收紧**：给 CC 的任务控制在 1-2 个文件改动
- **自动复盘**：每次 PR 提出后做结构化复盘（planned vs actual、踩坑、是否值得沉淀 skill）
- **ANTI-PATTERNS**：给自己定义"不该做什么"清单
- **借鉴→落地流程**：系统化吸收外部知识，不是看完就忘

### 需要积累的
- **领域专家 Skill**：等有更多 coding 经验后，按项目类型建专家知识库
- **三方审查**：目前是单 agent，未来多 agent 时适用
- **工作流声明化**：先用 skill 模拟，成熟后考虑 YAML DAG

### 不适用于我们
- 12 agent 并行编码的冲突管理（我们只有 1 个 CC）
- DISPATCH-BOARD 调度板（单 agent 不需要）
- 代码改动工具约束（我们已经统一走 CC）
